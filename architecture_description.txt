========================================
АРХИТЕКТУРА ИНТЕГРАЦИИ HRM СИСТЕМЫ
========================================

1. ОБЩАЯ СХЕМА
----------------------------------------
Система состоит из 3 микросервисов и интеграционного слоя:

┌─────────────────────────────────────────────────────────────────┐
│                         WEB UI (Port 8080)                       │
│                      Bootstrap + JavaScript                      │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                   ИНТЕГРАЦИОННЫЙ СЕРВИС                          │
│              (Оркестратор + Адаптеры + Saga)                     │
└─────────────────────────────┬───────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│   МОДУЛЬ А    │    │   МОДУЛЬ Б    │    │   МОДУЛЬ В    │
│  Сотрудники   │    │   Вакансии    │    │    Заявки     │
│   Flask:5001  │    │ FastAPI:5002  │    │  Flask:8000   │
└───────┬───────┘    └───────┬───────┘    └───────┬───────┘
        │                    │                    │
        └────────────────────┼────────────────────┘
                             ▼
              ┌─────────────────────────┐
              │   SQLite / PostgreSQL   │
              │    (единая БД)          │
              └─────────────────────────┘

2. КОМПОНЕНТЫ ИНТЕГРАЦИИ
----------------------------------------
- Адаптеры: преобразование форматов данных между модулями
- DTO: объекты передачи данных
- State Machine: управление статусами заявки
- Saga: компенсационные транзакции при ошибках
- Event Bus: шина событий для асинхронной коммуникации

3. ТЕХНОЛОГИИ
----------------------------------------
- Python 3.11
- Flask (модули А и В)
- FastAPI (модуль Б)
- SQLAlchemy (ORM)
- SQLite / PostgreSQL (БД)
- Bootstrap 5 (UI)
- GitHub Actions (CI/CD)

4. ВЗАИМОДЕЙСТВИЕ МОДУЛЕЙ
----------------------------------------
Сотрудник (UI) → Интеграционный сервис → Модуль А (проверка)
                → Модуль Б (проверка)
                → Модуль В (создание заявки)
                → БД (сохранение)
