========================================
ДЕФЕКТЫ HRM СИСТЕМЫ
========================================

ДЕФЕКТ #1: Ошибка 404 при GET /vacancies/{id}
----------------------------------------
Причина: В модуле Б отсутствовал эндпоинт для получения одной вакансии
Решение: Добавлен @app.get("/vacancies/{vacancy_id}")
Статус: ИСПРАВЛЕН

ДЕФЕКТ #2: Интеграционный сервис не находил сотрудника
----------------------------------------
Причина: Использовался localhost вместо 127.0.0.1 в URL
Решение: Заменены все URL на 127.0.0.1
Статус: ИСПРАВЛЕН

ДЕФЕКТ #3: Pydantic предупреждения о deprecated dict()
----------------------------------------
Причина: Использовался устаревший метод .dict()
Решение: Заменён на .model_dump()
Статус: ИСПРАВЛЕН

ДЕФЕКТ #4: ModuleNotFoundError в тестах
----------------------------------------
Причина: Отсутствовал sys.path для импорта модулей
Решение: Добавлен sys.path.insert(0, ...)
Статус: ИСПРАВЛЕН

ДЕФЕКТ #5: Ошибка кодировки эмодзи в Windows
----------------------------------------
Причина: Windows консоль не поддерживает эмодзи
Решение: Убраны эмодзи, добавлена UTF-8 кодировка
Статус: ИСПРАВЛЕН
